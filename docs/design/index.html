SAND: Raiders of Sophie — Wiki

Approved high-fidelity visual reference for the shadcn/ui rebuild. Every screen is shown at desktop and mobile width with its key states (hover, active filter, empty, loading). These are framework-free HTML/CSS mockups — precise about color, type, spacing, density, and component states — not production React.

Direction: polished chrome (sticky blurred nav, hero, cards — rusthelp) wrapped around dense, utilitarian data grids and stat tables (rustlabs). Identity: SAND desert / orange. Dark only. Squared corners. Oswald display.

App shell

Sticky blurred header — brand mark, section-dropdown nav, global search, auth menu — over a quiet restyled footer. On mobile the nav collapses to a hamburger that opens a left slide-in drawer. The blur + 82%-opacity background is the only place transparency is used.

Deviation  The navbar-search brainstorm placed search inline in the bar at all widths; I keep it inline on desktop but move it into the drawer on mobile so the bar holds only brand + hamburger + auth at narrow widths.

SCREEN · 03

Items list

[ sidebar | grid ]. Sticky category sidebar with per-category counts, a row of rarity filter chips, and a responsive EntityCard grid. The EntityCard is the workhorse: rarity rail + sprite tile + name/type + right-aligned stats. On mobile the sidebar collapses to a horizontal chip row that opens a category drawer.

Deviation  The cards.html round used a full rarity-tinted card background; the rarity-filter round used a thin rail. I unified on the 4px rail + neutral card (keeps AA contrast on names and lets the grid stay calm) and reserve full rarity color for the badge + detail header.

SCREEN · 04

EntityDetail

Large sprite + rarity badge + category badge header, a prominent stat grid, tabbed relationship tables (crafting / used-in / loot), and a Details facts panel. Adaptive layout: when an entity has relationship rows it uses a [ main | facts sidebar ] split; entities with no relations fall back to a single centered column (shown as a state below).

Deviation  The detail-stats round stacked stats as a vertical list; I promoted them to a bordered stat grid (rustlabs density) so the headline numbers read first. Crafting / Used-in / Loot were three separate panels in the brainstorm — unified into one tabbed table here.
